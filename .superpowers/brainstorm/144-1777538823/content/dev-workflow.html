开发 / 测试 / 部署 三环境流水线

本地写代码 → 手机验证 → 服务器运行

环境角色分配

💻 Windows 本地

写代码、改代码
Vite Dev Server 跑前端
可用 VS Code 远程编辑服务器文件
不跑 Redis/MQTT/MinIO

📱 Android 手机

Termux 运行 phone-agent
浏览器打开前端页面
ADB/无障碍服务触控
抖音/快手/小红书 App 发布

🖥️ Linux 服务器

Docker Compose 全栈
OpenClaw + Redis + MinIO
Mosquitto MQTT
creator-api :3001

核心原则

1
所有有状态服务只跑在服务器上

Redis、MQTT、MinIO、OpenClaw 都在服务器 Docker 里，本地不装

2
creator-frontend 双模式

开发时跑在 Windows（Vite热更新），发布时构建到 dist 由服务器 serve

3
creator-api 更新靠 Docker 重新构建

改完代码 push 到服务器 → docker compose build creator-api → docker compose up -d

场景一：本地开发前端 + 服务器后端（日常）

日常改 UI 时的通信链路
# 1. 修改 vite.config.ts 的 proxy 指向服务器
# 创建 .env.development 文件：
VITE_API_TARGET=http://192.168.x.x:3001

# 2. Windows 启动前端 dev server
$ cd creator-frontend
$ npm run dev  → http://192.168.x.x:5173

# 3. 手机浏览器打开
http://192.168.x.x:5173
   └─ /api/* ──proxy──▶ 服务器:3001
                 └─ OpenClaw:3000

✨ 好处：改完前端代码自动热更新，手机立刻看到效果

场景二：改 creator-api 后端代码

改完 API 代码后更新服务器
# 1. Windows 本地改代码
$ vim creator-api/routes/messages.js

# 2. 推送到服务器（git push 或 rsync）
$ git push
$ ssh user@server
$ cd /opt/avatar-ai-video
$ git pull

# 3. 重新构建并重启
$ docker compose build creator-api
$ docker compose up -d creator-api

⏱ 每次改 API 需要几十秒重建，不算重

场景三：测试手机 Agent

手机端调试 ADB/无障碍操作
# 1. 手机 Termux 里启动 agent
$ cd phone-agent
$ MQTT_BROKER=mqtt://server-tailscale-ip:1883 \
  PHONE_ID=phone_01 \
  PLATFORM=douyin \
  node agent.js

# 2. 看到日志
[MQTT] connected to server
[STATUS] online, battery=85%

# 3. 服务器手动发测试任务
$ docker exec -it mosquitto mosquitto_pub \
  -t phone/phone_01/task \
  -m '{"task_id":"test","actions":[{"type":"screenshot","name":"test"}]}'

# 4. 手机 Agent 收到任务 → 执行 → 回传状态
[TASK] test - 发布成功

💡 手机端改代码：用 VS Code Remote SSH 连 Termux 直接编辑

场景四：完整端到端测试

# 前置条件：服务器全栈运行、手机 agent 在线

# 手机浏览器打开前端
http://server-ip:3001

# → 创建会话 → 4轮对话 → 确认提交
# → creator-api → OpenClaw → RunningHub Skill
# → 视频生成 → MinIO 存储 → Dispatch Skill
# → MQTT 下发 → 手机 Agent 收到 → 下载视频
# → ADB 操作抖音/快手/小红书 App → 发布成功

关键配置：让 Vite 代理指向服务器

修改一个文件，让手机访问本地前端时 API 自动走服务器

// creator-frontend/vite.config.ts
export default defineConfig({
  plugins: [react()],
  server: {
    host: '0.0.0.0',  // 允许局域网访问
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://你的服务器IP:3001',
        changeOrigin: true,
      }
    }
  }
});

一句话总结

Windows 只跑 Vite（前端热更新）→ 服务器 跑全部后端 → 手机 浏览器 + Termux Agent

三者通过局域网 / Tailscale 互联，开发改代码不需要部署到服务器，只有 API 改动才需要 docker compose rebuild
