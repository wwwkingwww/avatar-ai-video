Web 前端 vs Flutter — 不冲突

两种方案是互补关系，不是替代关系

项目中的"两个客户端"

📱 创作者端（我们现在在设计的）

用途：自媒体创作者提交视频需求

• 打开手机浏览器访问

• 对话式收集需求（4轮）

• 上传素材

• 提交到 OpenClaw 执行

React / Vue / HTML 任选
🤖 手机 Agent 端（已有 Node.js）

用途：在 Android 手机上自动操作 App

• 接收 MQTT 任务

• 模拟触控发布视频

• 截图回传服务器

• 心跳上报状态

Node.js (Termux) / 未来 APK

与 Flutter 的关系

✅
完全不冲突

Web 前端（React/Vue/HTML）和 Flutter App 都是创作者端的两种实现方式，后端 API 是同一套。就像同一个后台可以同时有 Web 版和 App 版。

🔄
渐进式：先 Web 后 Flutter

先用 Web（React/Vue）快速上线验证整个流程，后期如果创作者需要更好的移动端体验，再用 Flutter 做一个原生 App 替换前端层。后端 Server 和 API 一行都不用改。

📋
项目里已有的 Flutter 规划

现有文档 phone-agent-apk-design.md 规划的是手机端 Agent 的 APK 方案（替代 Termux+Node.js），运行在发布手机上。这不是创作者端 UI，是另一个子系统。

推荐路径

第一阶段 → 第二阶段
🌐
Phase 1
Web 前端
React/Vue/HTML
+ Node.js Server
→
📱
Phase 2（可选）
Flutter App
替换 Web 前端

结论：选 React / Vue 做 Phase 1，与未来 Flutter 完全不冲突。
后端 API 复用，前端层可随时替换。
