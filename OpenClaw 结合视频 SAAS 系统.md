**OpenClaw 承担的核心工作：**
1. **内容策划** — 根据用户的账号定位、历史数据、热点趋势，自动生成选题和脚本
2. **工作流编排** — 串联「选题 → 文案 → AI生成/渲染 → 后期 → 发布」全链路
3. **发布执行** — 利用 OpenClaw 的浏览器控制能力，模拟登录各视频平台、上传视频、填写标题/标签/封面、定时发布
4. **数据回收** — 发布后定期抓取播放量、点赞、评论等数据，反馈给策划 Agent 形成闭环
5. **异常处理** — 上传失败、账号异常时的重试和通知

**优势：** OpenClaw 能力利用最充分，整个流程有统一编排
**挑战：** 多租户隔离需要设计；OpenClaw 本身是单用户设计，需要包装

**手机只需要：**

**OpenClaw 通过 ADB 从服务器直接向手机发指令：**
```bash
adb -s 192.168.1.10:5555 shell input tap 500 800     # 点击
adb -s 192.168.1.10:5555 shell input text "视频标题"   # 输入文字
adb -s 192.168.1.10:5555 shell uiautomator dump        # 获取UI树
```

**但这里有个关键问题 ⚠️：**

原生 ADB 只能做「坐标点击」，换个分辨率/系统版本就挂了。**ClawMobile 的核心价值**就是把「语义识别」（点击"发布"按钮）的能力带到了手机上。

### 四、UI 定位策略（最关键的一环）

这是区别"能用"和"不能用的"的生死线：

```
层级 1: 语义定位 ⭐ 首选
→ Accessibility Service 读 UI 树
→ 找到文字是"发布"、contentDescription 是"上传视频"的元素
→ 直接获取其坐标 → 点击
→ 成功率最高，不依赖分辨率

层级 2: 半语义定位
→ uiautomator dump 导出 XML
→ 正则/模糊匹配："发.*布" 或 "下一步(.*[0-9])?"
→ 解析 bounds → 点击
→ 适合 Accessibility 被 App 主动隐藏的情况

层级 3: 结构定位（降级）
→ 按 UI 层级关系定位
→ "第三个 LinearLayout → 第二个 Button → 文本包含'发布'"
→ 适合非标准控件（WebView 内嵌、游戏引擎渲染）

层级 4: 坐标兜底（最后手段）
→ 预配置每个平台/版本的固定坐标映射
→ 只在以上都失败时使用
→ 需要维护坐标库，容易坏
```

### 六、手机 Agent 的代码骨架（伪代码）

```
手机端到底要跑什么？一个极简的 Node.js 脚本。
```

```javascript
// 手机 Termux 中运行：node agent.js
// 作用：连上服务器 → 等指令 → 执行 → 回传结果

// ====== 依赖 ======
// ws (WebSocket客户端)
// @devicefarmer/adbkit (或直接用 child_process 调 adb)

class PhoneAgent {
  constructor(config) {
    this.deviceId = config.deviceId;        // 唯一标识这台手机
    this.serverUrl = config.serverUrl;      // wss://your-server.com/agent
    this.accountMap = config.accounts;      // 这台手机管理的账号 {抖音: "acc_003", ...}
    this.screenReader = new ScreenReader();
    this.executor = new UIExecutor();
    this.reporter = new Reporter();
  }

  // ====== 生命周期 ======
  async start() {
    await this.connect();       // WebSocket 连服务器
    this.startHeartbeat();      // 10秒一次心跳
    this.listen();              // 等指令
  }

  // ====== 收到指令 ======
  async listen() {
    this.ws.on('message', async (raw) => {
      const cmd = JSON.parse(raw);

      // 服务器发来一个完整发布序列
      if (cmd.type === 'sequence') {
        for (const step of cmd.steps) {
          const result = await this.executor.run(step);
          this.reporter.send(cmd.id, step, result);

          if (result.status === 'failed') {
            // 把失败步骤 + 截图回传服务器，让 AI 决策下一步
            await this.reportFailure(cmd, step, result);
            break;
          }
        }
      }
    });
  }

  // ====== 手机当前状态快照 ======
  async getState() {
    return {
      currentApp: await this.getForegroundApp(),
      uiTree: await this.screenReader.dumpAccessibility(),
      screenshot: await this.screenReader.capture(),
      battery: await this.getBattery(),
      network: await this.getNetworkType()
    };
  }
}
```

手机 Agent 核心就 **200-300 行** Node.js，不跑 AI 模型，不存储业务逻辑。

### 七、服务器端的 OpenClaw Skill 长什么样

手机 Agent 只管执行，那"发布视频到抖音"这个流程的编排全部在服务器 OpenClaw 上：

这个 Skill 内部逻辑：
1. 检查账号状态（是否在线、是否被限流）
2. 从文件存储拉取视频文件
3. 生成标题、话题标签（AI）
4. 推送到目标手机的 Agent
5. 下发完整序列指令（见上面协议设计）
6. 等结果 → 成功则记录日志，失败则重试或通知人工
7. 发布成功后，设置定时任务，1小时后回收播放数据

## 几个关键点逐个说

### 1. 多租户隔离 — 纯存储层的事

```
最简单方案：S3/OSS 目录前缀隔离

s3://your-bucket/
├── tenant_001/
│   ├── uploads/          ← 用户上传的原始素材
│   │   ├── video_01.mp4
│   │   └── image_cover.jpg
│   ├── outputs/          ← AI处理后的成品视频
│   │   └── video_01_final.mp4
│   └── temp/             ← 处理中的临时文件
├── tenant_002/
│   └── ...
```

- 安全靠 **服务端鉴权**，用户永远拿不到 signed URL 以外的访问
- 存储成本不贵，国内 OSS 约 ¥0.12/GB/月
- 隔离天然隔开，不存在串数据风险

### 2. 大文件上传 — 现成轮子

手机端上传 200MB+ 的视频，需要用分片上传 + 断点续传：

```javascript
// 手机 App 端（伪代码）
// 不需要自己写，现成的 SDK：

// 阿里云 OSS
import OSS from 'ali-oss';

const client = new OSS({
  region: 'oss-cn-hangzhou',
  accessKeyId: 'STS.Credentials',
  accessKeySecret: 'STS.Credentials',
  stsToken: 'STS.Credentials',     // 从你的后端获取临时Token
  bucket: 'your-video-bucket',
});

// 分片上传，自带断点续传
await client.multipartUpload(`tenant_001/uploads/video.mp4`, filePath, {
  progress: (p) => {
    console.log(`上传进度: ${p * 100}%`);
  },
});
```

- OSS SDK 原生支持分片上传 + 断点续传
- 进度条回调直接给了
- 不需要自己写，一行配置搞定

### 3. 安全链路 — STS 临时凭证模式

```
   App                         后端                        OSS
   │                            │                          │
   │  ① 请求上传凭证             │                          │
   │  POST /api/upload/init     │                          │
   ├───────────────────────────►│                          │
   │      body: {               │                          │
   │        fileName: "xx.mp4", │                          │
   │        fileSize: 200MB     │ ② 校验租户是否有权限       │
   │      }                     │    校验文件名/类型/大小     │
   │                            │    生成STS临时Token        │
   │  ③ 返回临时凭证 + 路径       │    (有效期15分钟)         │
   │◄───────────────────────────│                          │
   │  {ossKey, stsToken,        │                          │
   │   ossEndpoint,             │                          │
   │   filePath: "tenant_001/   │                          │
   │     uploads/xx.mp4"}       │                          │
   │                            │                          │
   │  ④ 直接用STS Token直传OSS   │                          │
   ├──────────────────────────────────────────────────────►│
   │  上传完成，OSS回调           │                          │
   │                            │                          │
   │  ⑤ 通知后端上传完成          │                          │
   ├───────────────────────────►│                          │
   │                            │ ⑥ 写入任务队列            │
   │                            │    通知 OpenClaw          │
```

**好处：**
- 文件不经过你的后端服务器（省带宽、省机器）
- STS Token 临时有效，泄露也没用
- 权限精确到 `tenant_001/uploads/*`，别人碰不到

```
  多台手机同时发布不同视频
      ↓
  同步下载？不行，太慢且占存储

解决：
  ┌─────────────────────────────────────┐
  │  手机 Agent 内置一个文件下载管理器    │
  │                                     │
  │  1. 收到发布任务                    │
  │  2. 从 S3 预签名 URL 下载视频        │
  │  3. 调用 ADB 发给抖音               │
  │  4. 发布成功后立刻删除本地文件        │
  │  5. 存储低于 500MB 时拒绝新任务      │
  │                                     │
  │  额外处理：                         │
  │  · 视频下载中也能先做前置操作        │
  │  · 支持边下边推（流式）             │
  │  · WiFi-only 模式（4G/5G 流量贵）   │
  └─────────────────────────────────────┘
```
